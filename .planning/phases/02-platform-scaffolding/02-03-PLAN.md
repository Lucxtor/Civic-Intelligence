# Plan 02-03: Civic Onboarding Modal

## Goal
Build the full-screen glass onboarding modal that appears for new wallet addresses. The user selects their demographic group per dimension and the data is saved via `/api/user/profile` (POST) → Prisma → SQLite.

## Tasks

<task id="02-03-A">
<action>
Create `src/components/auth/OnboardingModal.tsx`. This is a full-screen blocking overlay using the existing glass-panel design language. It reads `showOnboardingModal`, `dimensions`, and `walletAddress` from `useProfileStore`.

Full component structure:
- Fixed dark backdrop: `fixed inset-0 z-50 flex items-center justify-center bg-black/80 backdrop-blur-md`
- Central card: `glass-panel max-w-lg w-full mx-4 p-10 rounded-3xl relative overflow-hidden`
- Top gradient bar: `absolute top-0 left-0 w-full h-1 bg-gradient-to-r from-ipe-green via-ipe-yellow to-ipe-magenta`
- Framer Motion entry: `initial={{ opacity: 0, y: 40 }} animate={{ opacity: 1, y: 0 }} transition={{ type: 'spring', damping: 25 }}`
- Heading: **"Welcome. To participate in governance, we need your civic context."**
- Subtitle: `"Your selections are stored off-chain and used only to enrich anonymous community analytics."`
- Dynamic form: `dimensions.map(dim => <label> + <select>)` with the dim's options as `<option>` elements. Each select value tracked in local React state (`Record<string, string>`).
- Submit button: `"Join the Civic Network"` — `bg-ipe-green text-black font-bold`
- Validation: show inline red error `"Please complete all fields."` if any dimension is unselected on submit.

On submit:
1. Validate all dimensions are selected.
2. `fetch('/api/user/profile', { method: 'POST', body: JSON.stringify({ walletAddress, demographics: selectedValues }) })`.
3. On success (`res.ok`): call `useProfileStore().setProfile(selectedValues)` to mark onboarded and dismiss modal.
4. On error: show inline error message, stay on modal.
</action>
<acceptance_criteria>

- On success: calls `setProfile(demographics)` which sets `isOnboarded: true` and `showOnboardingModal: false`.
- Uses fixed overlay, `glass-panel`, gradient top bar, and Framer Motion entry.
- Exact heading: *"Welcome. To participate in governance, we need your civic context."*
</acceptance_criteria>
</task>

## Verification
- Navigate to app with a new (never-connected) wallet address.
- Connect wallet → onboarding modal appears immediately (full-screen, blocking).
- All seeded demographic dimensions appear as labeled dropdowns.
- Submitting with all fields selected → `GET /api/user/profile?address=0x...` now returns the user (not null).
- Reconnecting the same wallet later → modal does NOT appear.

## Must Haves
- Modal must be blocking — `pointer-events-none` or `overflow-hidden` on the body when the modal is open.
- Must show the exact heading copy specified above.
- Error state must stay on the modal (do not close on API error).
