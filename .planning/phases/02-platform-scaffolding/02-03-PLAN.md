# Plan 02-03: Civic Onboarding Modal

## Goal
Build the full-screen glass onboarding modal that appears for new wallet addresses. The user selects their demographic group per dimension and the data is saved to Supabase `profiles`.

## Tasks

<task id="02-03-A">
<action>
Create `src/components/auth/OnboardingModal.tsx`. This is a full-screen overlay (fixed, z-50) using the existing glass-panel design language.

Structure:
- Fixed dark backdrop (`bg-black/80 backdrop-blur-md`)
- Central glass-panel card (`max-w-lg w-full p-10 rounded-3xl`)
- Top color bar: `bg-gradient-to-r from-ipe-green via-ipe-yellow to-ipe-magenta h-1`
- Heading: "Welcome. To participate in governance, we need your civic context."
- Subtitle: "Your selections are stored off-chain and used only to enrich anonymous community analytics."
- A dynamic form: for each `DemographicDimension` in `useProfileStore().dimensions`, render a labeled `<select>` dropdown with the dimension's options.
- Submit button: "Join the Civic Network" — styled with `bg-ipe-green text-black font-bold`.
- Framer Motion: slide-up entry (`initial={{ opacity: 0, y: 40 }} animate={{ opacity: 1, y: 0 }}`).

On submit:
1. Validate all dimensions are selected (show inline error if not).
2. Call `supabase.from('profiles').insert({ wallet_address: walletAddress, demographics: selectedValues })`.
3. On success: call `useProfileStore().setProfile(savedProfile)` to dismiss modal and mark user as onboarded.
4. On error: show error message inline (do not dismiss modal).
</action>
