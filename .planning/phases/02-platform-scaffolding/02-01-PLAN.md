<task id="02-01-B">
<action>
Initialize Prisma and create `prisma/schema.prisma` with the full data model. Use the SQLite provider.

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

// A connected wallet address and their self-reported demographic profile.
model User {
  walletAddress String   @id @map("wallet_address")
  demographics  String   // JSON string: { "ageGroup": "25–34", "occupation": "Tech Worker", ... }
  createdAt     DateTime @default(now()) @map("created_at")

  @@map("users")
}

// Admin-managed demographic dimensions.
// e.g. { key: "ageGroup", label: "Age Group", options: ["18–24", "25–34", ...] }
model DemographicDimension {
  id        String   @id @default(cuid())
  key       String   @unique
  label     String
  options   String   // JSON string: array of option strings
  createdAt DateTime @default(now()) @map("created_at")

  @@map("demographic_dimensions")
}

// Anonymized vote — wallet address is NEVER stored here.
model AnonymizedVote {
  id          String   @id @default(cuid())
  proposalId  String   @map("proposal_id")
  demographics String  // JSON string: demographic tags only, no wallet
  responses   String   // JSON string: { "fairness": 4, "feasibility": 3, ... }
  createdAt   DateTime @default(now()) @map("created_at")

  @@map("anonymized_votes")
}
```
</action>
<acceptance_criteria>
- `prisma/schema.prisma` exists with `User`, `DemographicDimension`, and `AnonymizedVote` models.
- `datasource db` uses `provider = "sqlite"` and `url = "file:./dev.db"`.
- `AnonymizedVote` has NO `walletAddress` field.
</acceptance_criteria>
</task>

<task id="02-01-C">
<action>
Create the Prisma Client singleton at `src/lib/db.ts`. This prevents connection pool exhaustion during Next.js hot-reloading in development:

```typescript
// src/lib/db.ts
import { PrismaClient } from '@prisma/client';

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const db =
  globalForPrisma.prisma ??
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
  });

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = db;
```
</action>
<acceptance_criteria>
- `src/lib/db.ts` exists and exports `db` (a PrismaClient instance).
- Uses `globalThis` singleton pattern.
</acceptance_criteria>
</task>

<task id="02-01-D">
<action>
Create a seed script at `prisma/seed.ts` to populate default `DemographicDimension` rows:

```typescript
// prisma/seed.ts
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
  const dimensions = [
    {
      key: 'ageGroup',
      label: 'Age Group',
      options: JSON.stringify(['18–24', '25–34', '35–44', '45–54', '55+']),
    },
    {
      key: 'occupation',
      label: 'Occupation',
      options: JSON.stringify([
        'Tech Worker',
        'Student',
        'Educator',
        'Public Servant',
        'Business Owner',
        'Other',
      ]),
    },
    {
      key: 'district',
      label: 'District',
      options: JSON.stringify([
        'Centro',
        'Trindade',
        'Lagoa',
        'Campeche',
        'Continente',
        'Other',
      ]),
    },
  ];

  for (const dim of dimensions) {
    await prisma.demographicDimension.upsert({
      where: { key: dim.key },
      update: {},
      create: dim,
    });
  }

  console.log('✓ Seed complete — default demographic dimensions created.');
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect());
```

Add the following to `package.json`:
```json
"prisma": {
  "seed": "ts-node --compiler-options {\"module\":\"CommonJS\"} prisma/seed.ts"
}
```

Also install `ts-node`:
```bash
npm install -D ts-node
```

Then run:
```bash
npx prisma db push
npx prisma db seed
```
</action>
<acceptance_criteria>
- `prisma/seed.ts` exists with upserts for `ageGroup`, `occupation`, and `district` dimensions.
- `package.json` has a `"prisma": { "seed": "..." }` field.
- Running `npx prisma db push` creates `prisma/dev.db`.
- Running `npx prisma db seed` populates the `demographic_dimensions` table.
</acceptance_criteria>
</task>
