## Goal
Scaffold the high-fidelity shell of the Civic Intelligence Platform. This phase moves beyond simple setup to create the core user experience across all major functional routes.

## Design Philosophy: "The Techno-Optimist Forge"
- **Aesthetic**: Blending bioluminescent Web3 accents with glassmorphism panels.
- **Typography**: Space Grotesk for headers (structure/logic) and Inter for UI copy (readability).
- **Architecture**: RESTful App Router structure (`/admin`, `/proposals`, etc.).

## Detailed Component Concepts

### 1. The Admin Proposal Forge (`/admin`)
The "source of truth" for civic governance.
- **Form Layout**: 
  - **Left Column**: Registry Schema Inputs (Title, Abstracts, Full Markdown).
  - **Right Column**: Live Impact Matrix Preview. 

#### Structured Impact Matrix
- **Beneficiaries**: Array of `{ tag: string, estimate: number }`. 
  - *UI Experience*: Tag-cloud style inputs with "Estimated Reach" mini-sliders for each group.
- **Negative Impacts**: Array of `{ tag: string, risk: 'Low' | 'Moderate' | 'Severe' }`. 
  - *UI Experience*: Multi-select chips with color-coded risk indicators.
- **Financials**: Object `{ total_cost: number, currency: string, roi_confidence: number }`.
  - *UI Experience*: Currency-formatted inputs with a "Precision Confidence" radial gauge.

#### Live Preview: The Governance Radar
- On the right side of the Admin Forge, a **Recharts Radar Chart** visualizes the metadata in real-time.
- Axes: `Sustainability`, `Economic Growth`, `Social Equity`, `Innovation`.
- This ensures admins have a "holistic" view of the proposal before broadcasting it to the community.

### 2. Proposal Discovery & Navigation Flow
The entry point for community participation.
- **Active Navigation**: A persistent **Breadcrumb** appears in the header once a proposal is selected (e.g., `Home > Precision Tax Rebalancing > Nuance Vote`).
- **Interactive Bento**: Proposal cards feature a "Technical Sparkline" on hover, hinting at the complexity/activity of the proposal.

### 3. Detailed Proposal View & Vote flow
The transition from reading to deciding.
- **Summarization Toggle**: A polished layout-switch between ELI5 (Plain language) and SME (Technical) abstracts. 
- **The Nuance Slider (Vote Form)**: Instead of a binary Yes/No, users manipulate bioluminescent sliders for each metric defined by the Admin.
  - **Approve/Disapprove**: The only mandatory metric.
  - **Custom Facets**: (e.g., Economic Impact, Feasibility) are optional sliders for deep feedback.

### 4. Demographic Sentiment Dashboard (`/proposals/[id]/dashboard`)
Visualizing the "Civic Soul".
- **Demographic Clusters**: Grid-based visualizations of sentiment cross-referenced with demographic slices (Age, Economy, etc.).
- **Privacy Core**: Threshold revealing (groups <10 are blurred or aggregated).

## Technical Strategy
- **Routing**: Next.js RESTful conventions.
- **Persistence**: Zustand with `persist` middleware for a robust "Backend-less" demo.
- **Auth Simulation**: RainbowKit/Wagmi initialized to provide the source of demographic data.
- **State**: Centralized mock state that handles dynamic metric generation.
